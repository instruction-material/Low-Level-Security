# LLS13 Crash Classification Lab

Practice turning a toy-program failure into an exploitability note for a
maintainer.

## Goals

- distinguish denial of service, disclosure, and corruption outcomes
- reason about reachability and attacker control without exaggeration
- write a short exploitability note grounded in evidence

## Suggested workflow

1. Read each toy report and classify its likely impact.
2. Explain which assumptions matter most: reachability, controllable data, and
   mitigation coverage.
3. Compare the starter's rough classification to the reference solution.
